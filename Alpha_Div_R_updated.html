This tutorial will guide you through plotting the alpha diversity data generated by QIIME and testing for significant differences.

What is Alpha Diversity

Alpha diversity was defined by Whittaker (1972) as the species richness of a place. However, the practical development of this concept led us to redefine alpha diversity on the basis of the structure of the community. The most common expression is linked with both on the number of species (e.g. ‘observed species’) and the proportion in which each species is represented in the community. The amount of diversity in any community is extremely important in determining ecological dynamics (e.g. community productivity, stability, and resilience). For humans, there is a great deal of data demonstrating the ancestral human gut microbiome is more diverse than the modern one, and that this lower diversity is highly correlated with a number of important diseases. Therefore, alpha diversity is an important phenotype in microbiome research.

We used alpha diversity earlier to look at different rarefaction depths. Now that we have rarefied or filtered our OTU table, we can find out the alpha diversity of each of our samples.

Inputs

Diversity File

Prior to plotting in R, we need to generate an alpha diversity table in QIIME. This file will be the output of alpha_diversity.py, and will be a tab-delimited, plain text file. The format for the alpha diversity file is the following:

Column 1 Sample IDs

Columns 2 through N Each column represents an diversty metric (e.g. ‘PD_whole_tree’, ‘simpson’, ‘shannon’, or ’observed_species. The numbers in each row correspond to alpha diversity estimate for the associated sample.

Metadata

Your metadata file (also called a mapping file) is a data table containing information about the samples in your dataset. In order to plot alpha diversity grouped by a variable of interest (e.g. body site, sex, eco-region, BMI, etc.), you need to have that information about your samples accessible. The metadata file for our data set is HMP_5BS_metadata.txt.

Column 1

The header for this column is '#SampleID, and as the name implies it lists the samples IDs.

Columns 2 through N

Each column represents a descript of the sample. It can be anything including details about the patient, person, animal or location the sample was taken from.

Plotting

Box Plots

There are many different ways to plot these types of data. Box plots are a convenient way to graphically depict groups of numerical data. Box plots consist of several features: the box, which extends to the first quartile (Q1) to the third quartile (Q3), respectively, with the median (Q2) depicted by a vertical line within the box; whiskers, which extending vertically from the box and indicate the range of variability outside of the upper and lower quartiles; and outliers, which are individual points outside of the box and whiskers.

It is important to note that box plots are non-parametric. They display variation in samples of a statistical population without making any assumptions of the underlying statistical distribution. The spacing between the interquartile range of the box indicates the degree of dispersion (spread) and skewness in the data.

Testing for Differences

t-Tests

A t-test is a statistical hypothesis test that can be used to determine if two sets of data are significantly different from each other based on the population means. It assumes a Student’s t-distribution (t-distribution), where as the sample size increases, the distribution of data approaches the bell shape of the standard normal distribution. Although it is typically assumed that data of a large enough sample size are normally distributed, this is not always the case.

Testing for Normality

Let’s see if our data are normally distributed using the hist() function. We will do this for each group in the covariate we are testing (in this case, 'sex').

#We will find the samples that are male in the metadata
males.ix <- metadata[,cov1] == 'male'
#And subset the alpha table to include only those
males <- as.data.frame(alpha[males.ix,])

#We will do the same for females
females.ix <- metadata[,cov1] == 'female'
females <- as.data.frame(alpha[females.ix,])

#Now we can plot the histograms
hist(females[,div_metric], xlab="Alpha Diversity", main='Females')


hist(males[,div_metric], xlab="Alpha Diversity", main='Males')

Do those look normally distributed? For the most part they do, but it is sometimes hard to tell. The best way to determine if your data are normally distributed is do a statistical test.

The Shapiro-Wilk Normality Test

This test can be run with the shapiro.test() function in R. It will generate an approximate p-value, which is adequate in assessing normality. In this case, p-values less than 0.1 are indicative of data that is significantly different from normal in its distribution. We will run this test for each group in our covariate of interest.

shapiro.test(females[,div_metric])

Mann-Whitney U Test

Based on these results, it is better to test for differences in our data using a statistical test that does not require normal distributions. The Mann-Whitney U test (aka Wilcoxen-rank_sum test) is smilar to the t-test. The null hypothesis of this test would be that that groups means differ only by chance. We can do a Mann-Whitney U test using the wilcox.test() function in R.

wilcox.test(females[,div_metric], males[,div_metric],  na.rm=TRUE)

## 
##  Wilcoxon rank sum test with continuity correction
## 
## data:  females[, div_metric] and males[, div_metric]
## W = 1016, p-value = 0.03003
## alternative hypothesis: true location shift is not equal to 0

#Note that you would run a t-test in the same way, only using the t.test() function

The p-value here is 0.109, which is greater than 0.05. Thus, the means of that we accept the null hypothesis that these two body sites are not significantly different for this metric of alpha diversity.

In our example we only have two groups, 'male' and 'female'. If we used a different covariate, like 'BodySite' (which contains 5 groups) we would have 10 tests to do. Luckily, for loops can come to the rescue! This loop will perform the wilcox.test() on every unique combination of groups in the covariate.

#First let's set all the groups available for the variable we care about
#In this case we will use BodySite instead of what we set as cov1 (because sex only has two values)
groups <- unique(metadata[,'BodySite'])

#We create empty vectors to store the pair-wise pvalues and the groups tested (names)
pw.pvalues <- NULL
pw.names <- NULL

#We set two counters, 'i' start at 1 and go until one less than the number of groups.  'j' will start at 2, and go until the full number of groups. This will end up comparing: 1 vs 2, 2 vs 3, 3 vs 4, and so on.
for(i in 1:(length(groups) - 1)){
            for(j in (i+1):length(groups)){
                ix.metric.i <- metadata[,'BodySite'] == groups[i] #we use this to pick the groups assigned to 'i' 
                ix.metric.j <- metadata[,'BodySite'] == groups[j] #and this for 'j'
                pvalue <- wilcox.test(alpha[ix.metric.i,div_metric], alpha[ix.metric.j,div_metric])$p.value #this stores the pvalue from the test
                pw.pvalues <- c(pw.pvalues, pvalue) #appends the new p-vale
                test.name <- paste(groups[i], "_vs_", groups[j],sep='') #sets the names of the groups tested
                pw.names <- c(pw.names, test.name) #appends the names of the groups tested
            }
}
names(pw.pvalues) <- pw.names

False Discovery Rate Correction

When we use the 'sex' covariate, we only have one test to perform. If we are comparing more than two groups we have to correct of the number of comparisons we are making. We do this with the p.adjust() function. This will correct for type I errors, which are rejections of a true null hypothesis (also known as a false positive).

fdr.pvalues <- p.adjust(pw.pvalues,'fdr')
